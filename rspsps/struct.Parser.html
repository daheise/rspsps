fn peek_n(&self, n: usize) -> String

Peek ahead at the next few characters in the stream, and return them. The The number of characters must be below the lookahead limit. End of file causes the returned string to be populated by EOF characters.

fn peek_str(&self, needle: &str) -> bool

Peek ahead and determine if the next characters in the stream are the given characters, in sequence. That is, the given string must be the next thing in the stream.

fn peek_str_and_consume(&self, needle: &str) -> bool

Peek ahead at the next few characters and if they are a given string, then consume them. Otherwise leave the stream unchanged.
